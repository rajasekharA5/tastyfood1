# 🚀 Netlify & GitHub Cleanup Guide

This guide explains how to delete duplicate or unused projects from your **Netlify Dashboard** and **GitHub Account** so you only keep your single, completed **Tasty Food** project.

---

## 🧹 1. Cleaning Up the Netlify Dashboard

If you have duplicate sites or test deployments on Netlify, follow these steps to delete them:

1. **Log in** to your [Netlify Dashboard](https://app.netlify.com/).
2. **Select the site** you want to delete from your site list.
3. Click on the ⚙️ **Site configuration** tab at the top.
4. Scroll all the way down to the bottom of the page to the **Danger zone** section.
5. Click the red 🗑️ **Delete this site** button.
6. A confirmation dialog will appear. Type the **name of the site** as prompted, then click **Delete**.

*Repeat this for any duplicate sites until only your main **Tasty Food** site remains.*

---

## 🐙 2. Cleaning Up your GitHub Repositories

If you have duplicate or test repositories on GitHub, here is how to delete them:

1. **Log in** to your [GitHub Account](https://github.com/).
2. Go to the main page of the repository you want to delete.
3. Click on the ⚙️ **Settings** tab (under the repository name).
4. Scroll all the way down to the bottom of the page to the **Danger Zone** section.
5. Click 🗑️ **Delete this repository**.
6. Select **I want to delete this repository**, then click **I have read and understand these effects**.
7. Type the **full name of the repository** (e.g., `username/repository-name`) to confirm.
8. Click the final **Delete this repository** button.
